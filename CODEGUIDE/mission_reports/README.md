# Mission Reports

Mission reports are periodic Soundemote progress snapshots.

They are not task instructions.

They are morale, state, and trajectory documents.

They may include:

* current project progress
* ASCII percentage bars
* active risks
* architecture status
* business/public status
* next strategic pressure points

Mission reports help Architect and Vision remember how far the system has moved.

They are especially useful during the 58-day push.

Suggested filename format:

```
YYYY-MM-DD_HHMM.md
```

Example:

```
2026-05-25_0856.md
```

Keep them honest.

Keep them motivating.

Keep them useful.
